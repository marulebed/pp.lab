# Лабораторная работа №5  
## Параллельное программирование (MPI, суперкомпьютер)

---

### Цель работы  
Исследовать зависимость времени выполнения программы умножения матриц от количества процессов при запуске на суперкомпьютере.

---

### Задание  
Модифицировать программу из лабораторной работы №3 и запустить её на суперкомпьютере с использованием технологии MPI. Провести эксперименты с различным количеством процессов.

---

### Описание программы  
Программа выполняет умножение квадратных матриц размером 1000×1000.  

Для распараллеливания используется технология MPI:
- данные распределяются между процессами  
- каждый процесс выполняет часть вычислений  
- результаты собираются в итоговую матрицу  

Запуск программы осуществляется с помощью batch-скрипта (SLURM), в котором задаётся количество процессов.

---

### Результаты экспериментов

|----------------|-----------------|------------|

### Анализ результатов  

С увеличением количества процессов время выполнения программы значительно уменьшается.  

При использовании 4 процессов вычисления выполняются быстрее более чем в 4 раза по сравнению с запуском на одном процессе. Это связано с тем, что задача делится между несколькими процессами, которые работают параллельно.  

Однако ускорение не является полностью линейным из-за накладных расходов на:
- передачу данных между процессами  
- синхронизацию  
- управление параллельными вычислениями  

---

### Вывод  

В ходе лабораторной работы была реализована параллельная версия программы с использованием MPI и выполнен запуск на суперкомпьютере.  

Проведённые эксперименты показали, что увеличение количества процессов позволяет существенно сократить время выполнения программы.  

Параллельные вычисления эффективно используются при работе с большими объёмами данных и позволяют значительно повысить производительность.
